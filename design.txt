DESIGN PATTERN: ATOMIC SPECULATIVE PARSING (SNAPSHOT & COMMIT)

This pattern handles ambiguous, "never-fail" markup (like Markdown) by 
treating the Tape as an atomic state that is only updated on a full match.

1. SNAPSHOT: At the start of a handler, create a local copy of `self.input`. 
   Since Tape is `Copy`, this is a cheap stack operation.
   
2. SPECULATE: Perform all lookahead and position advancing on the local 
   copy. Do not modify global state (like `self.tokens`) during this phase.

3. COMMIT: 
   - ON MATCH: If the syntax rules are satisfied, apply side effects 
     (e.g., `push_tok`) and overwrite `self.input` with the local copy.
   - ON FALLBACK: If rules aren't met, discard the local copy. The main 
     loop remains at the original position and can treat the current 
     character as plain text.

BENEFITS:
- Prevents "phantom" tokens from partial matches.
- Avoids borrow-checker conflicts (E0501) by removing the need for 
  closures that capture `self`.
- Provides zero-cost backtracking by simply letting the local copy 
  go out of scope.
